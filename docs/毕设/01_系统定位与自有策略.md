# 系统定位与自有策略

## 1. 课题边界

本毕设实现的是一套 **「黄金新闻 → 趋势强度 → 网格适配」** 的完整系统，核心贡献在 **新闻冲击如何改变网格交易行为**，而非复现某一现成商业/教学用全量回测引擎的全部细节。

| 层次 | 本系统（自有） | 外部参考（不依赖源码） |
|------|----------------|----------------------|
| 数据 | 金十/东财、宏观日历、FRED | — |
| 分析 | 词典/LLM 情感、五因子评分、surprise 方向 | — |
| **策略** | **GridAdapter + ShockTimeline（L1–L4）** | 老师侧 golden_shield 为独立网格底仓 |
| 执行 | MT5 FxPro 模拟盘 | — |
| 验证 | **自研 GridShockSimulator A/B** | gs_v17 对接为扩展项，非必需 |

老师要求「做属于我们的策略」，对应上表 **策略 + 验证** 两行；底仓可以是「对称间距网格 + 保证金模型」，冲击层由新闻驱动。

---

## 2. 自有策略定义

**名称：** 黄金新闻冲击网格适配策略（News Shock Grid Adapter）

**输入：** 宏观/快讯事件 → 方向（surprise / 动量 / 词典）、等级 L1–L4、持续小时数  

**输出（对网格的调整）：**

- `spacing_factor` — 扩大或收紧间距  
- `pause_reverse_orders` — 暂停反向挂单  
- `pause_all_new_orders` — L3 级停新开  
- `reverse_position_ratio` — 按比例平反向持仓  
- `directional_grid` — 冲击窗口内只挂顺势侧  

**规则实现：** `src/strategy_adapter/grid_adapter.py`、`src/backtest_adapter/policy.py`

---

## 3. 与 Baseline 的对比实验

- **Baseline：** 无 timeline，对称网格，相同 spacing/layers/资金/杠杆  
- **News Adapter：** 注入 `ShockTimeline`，在事件窗口内应用上述规则  
- **评价指标：** 终值权益、PnL、最大回撤、是否 stop_out、冲击动作次数  

**当前最优（H1，三年，$500，spacing 5.0×2 层）：** Run `202607121249`  
news **-51.25** vs baseline **-480.04**（baseline 强平，news 无强平）

## 4. 创新点表述（可直接进绪论/总结）

1. 宏观 **actual vs forecast surprise** 与 H1 动量融合的方向链，及品类阈值（CPI/NFP/PCE 分化）。  
2. **价格动量与新闻冲突** 时的冲击降级（`neutral_widen_only` 保留 L3 停单保护）。  
3. 小账户 **100x + 保证金 stop_out** 下的 risk overlay 与参数扫描（spacing/layers）。  
4. 新闻信号 → MT5 挂单调整的闭环（含 L2 删反向挂单验证）。
