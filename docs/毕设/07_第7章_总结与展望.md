# 第 7 章 总结与展望

---

## 7.1 工作总结

本文围绕黄金新闻实时分析与趋势强度判断，设计并实现了 **新闻冲击网格适配策略（News Shock Grid Adapter）** 及配套智能系统，主要完成：

1. 构建了从新闻抓取、NLP 情感、五因子评分到 GridAdapter 的实时分析链路，并与 MT5、Parquet 归档对接。  
2. 提出了基于 L1–L4 的分级冲击规则，以及 surprise→动量方向解析链与价格动量微调机制。  
3. 实现了自研 **GridShockSimulator**，在 $500、100× 杠杆、三年 H1 数据上完成 Baseline vs News 的 A/B 对比。  
4. 在 FxPro MT5 模拟盘上完成连接与 L2 删反向挂单的验证方案（截图待补）。

---

## 7.2 主要结论

1. **风险缓释优先：** 在小账户高杠杆场景下，对称网格 baseline 易触发强平（Run `202607121249`：baseline **-480.04** 且 stop_out）；news 组 **-51.25** 且 **无强平**，相对减亏 **428.79 USD**。  
2. **参数建议：** spacing **5.0**、max_layers **2** 较 3 层/更小间距更适合小账户；news 适配在参数扫描各组中均未强平。  
3. **Surprise 定位：** 方向命中率约 **50%**，CPI 子类略高；适合作为冲击防护触发，不宜单独作为盈利 alpha。  
4. **工程可扩展：** 引擎已支持点差与 M1 短窗 smoke，便于后续 realism 补充实验。

---

## 7.3 不足与局限

1. H1 bar 内事件时点精度有限，冲击激活时刻粗于 M1。  
2. 部分 macro forecast 来自 FRED 朴素预期，surprise 含噪。  
3. 历史回测以宏观日程为主，快讯词典方向样本少，与实时 loop 存在分布差异。  
4. 未完整建模佣金、隔夜利息；baseline 网格长期负期望，news 组改善的是 **相对风险** 而非保证绝对盈利。  
5. 未对接闭源全量第三方回测引擎，长周期 M1 全样本计算成本较高。

---

## 7.4 未来工作

1. **M1 长窗口正式回测：** 在 `m1_max_days` 约束下扩展至 30–90 天含点差实验。  
2. **L2/L3 实盘证据补全：** 交易时段完成 MT5 截图与日志归档，写入第 6 章。  
3. **Forecast 质量提升：** 接入一致预期或调查数据，提高 surprise 信噪比。  
4. **非宏观新闻历史回灌：** 扩充金十/东财历史标题，统一词典与 LLM 方向分布。  
5. **实盘小资金试运行：** 在模拟盘稳定后，评估更小资金与更严格 risk overlay 参数。

---

## 7.5 本章小结

本文实现了黄金新闻驱动的网格冲击适配系统，并通过三年 H1 回测表明其在防强平与减亏方面的有效性。后续将在 M1 realism、MT5 实证截图与 forecast 数据质量等方向继续完善。
