# 第 5 章 回测与实验

> **说明：** 本章为毕设正文初稿；实验数据来自 `E:\量化项目\05 Backtest\results\news-grid\`，Run ID 格式 `YYYYMMDDHHmm`。  
> **复现命令：** 见 §5.2 与 `docs/毕设/02_回测引擎说明.md`

---

## 5.1 实验目的

1. 验证 **新闻冲击网格适配策略** 相对 **无适配对称网格（Baseline）** 能否降低小账户在高杠杆下的极端亏损与强平概率。  
2. 比较不同网格间距与层数下 news 组的稳健性。  
3. 评估 surprise 方向规则对事件后金价短期走势的解释力。  
4. 检验自研引擎在 **点差** 与 **M1 分辨率** 下的可运行性（工程 realism，非替代 H1 长窗主结论）。

## 5.3 主实验：三年 H1 A/B 对比

### 5.3.1 最优参数 Run `202607121249`

在 PCE 降权、动量阈值 18、spacing 5.0×2 配置下：

**表 5-1 三年 H1 主实验结果（Run 202607121249）**

| 指标 | Baseline | News Adapter | 差值 |
|------|----------|--------------|------|
| 初始资金 | 500.00 | 500.00 | — |
| 终值权益 | 19.96 | **448.75** | — |
| 总 PnL | **-480.04** | **-51.25** | **+428.79** |
| 最大回撤 | 97.82% | **36.6%**（约） | 显著降低 |
| 是否强平 | **是** | **否** | — |
| 冲击动作 | 0 | 248 | — |

**分析：**

- Baseline 对称网格在三年 H1 样本上触发 **stop_out**，权益接近归零；说明 **$500 + 100× + 双侧叠层** 在无新闻防护时风险极高。  
- News Adapter 虽仍为负收益（-51.25），但 **避免强平**，相对 Baseline **减亏 428.79 USD**。  
- 在本课题定义下，策略的首要价值是 **宏观窗口内的风险缓释与防极端回撤**，而非保证三年正收益。

### 5.3.2 优化过程简要对比

**表 5-2 策略迭代关键 Run（news_adapter PnL）**

| Run ID | 主要改动 | news PnL | stop_out |
|--------|----------|----------|----------|
| 202607112147 | 扩展宏观日程 | -460.10 | Y |
| 202607120126 | claims 过滤 + L3 4h + risk overlay | -133.09 | N |
| 202607121058 | surprise 品类阈值 + FRED pct | -70.96 | N |
| **202607121249** | PCE 排除方向 + 动量18 + 5.0×2 | **-51.25** | N |

可见 **surprise 调优 + 网格降层 + 冲击窗口缩短** 对 small account 场景均有贡献。

## 5.4 参数敏感性：spacing × max_layers

固定信号与三年 H1 窗口，扫描 spacing ∈ {3.5, 4.0, 4.5, 5.0}、layers ∈ {2, 3}（Run 扫描记录 `202607121248`）。

**表 5-3 参数扫描（news 组，均未 stop_out）**

| spacing | layers | news PnL | vs baseline | news 最大 DD% |
|---------|--------|----------|-------------|---------------|
| **5.0** | **2** | **-206.64*** | +279.82 | 51.8 |
| 3.5 | 2 | -215.84 | +263.17 | 48.7 |
| 4.0 | 2 | -246.09 | +238.92 | 53.5 |
| 3.5 | 3 | -272.53 | +207.51 | 66.0 |
| 5.0 | 3 | -297.95 | +192.47 | 69.1 |

\* 扫描时部分信号配置与最终 `-51.25` Run 略有差异（动量阈值等）；趋势一致：**层数 2 优于 3，spacing 5.0 优于 3.5**。

**结论：** 小账户应优先 **减少叠层、增大间距**；news 适配在所有 8 组参数下均未强平，而 baseline 均强平。

## 5.5 Surprise 方向命中率实验

脚本：`scripts/eval_surprise_direction.py`  
方法：对 `_direction_source=surprise` 的信号，检查事件后 **4 根 H1** 金价变动是否与预测方向一致（|Δ| < 0.5 USD 记为 neutral 跳过）。

**表 5-4 Surprise 方向命中率（当前信号集）**

| 指标 | 数值 |
|------|------|
| 评估条数 | 122 |
| 命中数 | 61 |
| **命中率** | **50.0%** |
| CPI 子类 | 56.8% |
| NFP 子类 | 47.1% |
| PCE（方向仍参与评估的样本） | 偏低 → 已策略降权 |

命中率略高于随机（50%），CPI 表现最好；说明 surprise 规则 **具有一定短期解释力**，但不足以单独作为盈利 alpha，更适合作为 **冲击防护的触发条件**——与本章 A/B 结论一致。

## 5.6 点差与 M1 补充实验

### 5.6.1 点差模型

引擎支持 `spread_price=0.20` USD（GOLD 典型全价差）：  
开多成交价 = 限价 + 0.10；平多 = 标记价 − 0.10；空单对称。  
浮动盈亏按 exit 侧报价 mark，更贴近 CFD 多空单边报价。

H1 三年主结论表 5-1 为 **无点差** 条件下的 A/B 对照（与早期实验一致）；点差用于 **引擎 realism 补充**，可在附录给出「含点差 H1 14 天」对比。

### 5.6.2 M1 分辨率 smoke

| 项目 | 取值 |
|------|------|
| Run ID | 202607121344 |
| 窗口 | 14 天（smoke） |
| K 线数 | 14 036（M1） |
| 点差 | 0.20（默认开启于 config） |
| 结果 | 引擎跑通，baseline/news 在该短窗内信号稀疏时 PnL 可能相同 |

M1 全量三年约 350 万 bar，计算成本高；正式配置 `m1_max_days=30` 用于 **短窗 realism**，长窗结论仍以 **H1 三年** 为准。

**命令示例：**

```powershell

python scripts/run_news_grid_backtest.py --timeframe M1 --initial 500   # 最近 30 天
```

## 5.7 风险 overlay 与 margin 敏感性

`risk_overlay` 在 equity/used_margin 低于 80%/65% 时放大间距、减半手数与层数，并可在 emergency 档撤销挂单。  
敏感性实验（`202607120130_margin敏感性.md`）显示：在 news 路径部分参数段内 **overlay 未触发**（PnL  identical），说明本轮减亏 **主要来自 L2–L3 冲击规则与 directional grid**，而非 overlay 单独贡献。overlay 仍保留于 live 路径与极端 margin 场景。

---

## 5.8 实验局限

1. **H1 bar 内 shock 时点精度有限**，事件发生在 bar 内任意时刻均按 bar close 处理。  
2. **宏观 forecast** 部分来自 FRED 朴素预期（上期环比），非一致预期调查值，surprise 含噪。  
3. **历史回测几乎无 NLP 词典方向**（dict=0），与实时 loop 存在分布差异。  
4. **Baseline 网格本身长期负期望**，news 组改善的是 **相对风险**，非绝对盈利保证。  
5. **未含佣金、隔夜利息**；点差仅在补充实验启用。

---

## 5.9 本章小结

在 $500、100× 杠杆、三年 H1 黄金数据上，本文所提新闻冲击网格适配策略相对对称 Baseline **显著降低最大回撤并避免保证金强平**（Run `202607121249`，相对减亏 428.79 USD）。参数扫描表明 **spacing 5.0、2 层** 更适合小账户；surprise 方向命中率约 50%，支持其作为防护触发而非独立 alpha。点差与 M1 实验验证了自研引擎的可扩展性。下一章将给出 MT5 模拟盘上的执行层验证。
