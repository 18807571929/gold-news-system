# 自研回测引擎说明（GridShockSimulator）

## 1. 为何自研

- 毕设策略是 **新闻冲击适配层**，需在可控环境下做 Baseline vs Treatment A/B。  
- 不依赖老师 `golden_shield_v1.7` 源码，避免权限与复现边界问题。  
- 模块：`src/backtest_adapter/simulator.py`

## 2. 模型假设

| 项目 | 设定 |
|------|------|
| 合约 | 0.01 手 = 1 盎司，100 盎司/手 |
| 杠杆 | 默认 100x（config `backtest.leverage`） |
| 强平 | 保证金比例 ≤ 50% 全平（`stop_out_margin_level`） |
| 网格 | 限价多层挂单，间距 TP 平仓，对称或 directional |
| **点差** | 全价差 `spread_price`（默认 **0.20 USD**），买贵卖贱 |
| K 线 | **H1**（长期）或 **M1**（短期 realism，窗口受限） |

### 点差处理

- 开多成交价 = 限价 + spread/2  
- 开空成交价 = 限价 - spread/2  
- 平仓按 bid/ask：多单平于 mark - spread/2，空单平于 mark + spread/2  
- 浮动盈亏按 exit 侧报价 mark

## 3. 行情数据

| 周期 | 路径 |
|------|------|
| H1 | `02 History Data/data/XAUUSD_H1.csv` |
| M1 | `02 History Data/clean/XAUUSD_M1_clean.parquet` |

M1 全量约 **350 万** 根，正式回测默认 **最近 30 天**（`backtest.m1_max_days`）；发烟测试 14 天。

## 4. 运行命令

```powershell
# H1 三年正式
python scripts/run_news_grid_backtest.py --years 3 --initial 500

# H1 + 显式点差
python scripts/run_news_grid_backtest.py --years 3 --spread 0.20

# M1 发烟（约 2 周）
python scripts/run_news_grid_backtest.py --smoke --timeframe M1

# M1 正式（最近 30 天）
python scripts/run_news_grid_backtest.py --timeframe M1 --years 1
```

输出目录：`E:\量化项目\05 Backtest\results\news-grid\{run_id}\`

## 5. 局限性（论文中应写明）

- 简化限价撮合，无部分成交、无订单队列  
- 无隔夜利息、无佣金（仅点差）  
- H1  bar 内 shock 时间精度粗于 M1  
- Baseline 网格本身在小账户下易亏，adapter 主要体现 **减亏与防强平**
